The Problem

For small businesses or warehouses, managing inventory with manual methods or disconnected spreadsheets is inefficient and prone to error. It's difficult to get an accurate, real-time view of stock levels, leading to costly mistakes like over-ordering or running out of a critical item.

My Solution

I developed a dedicated, full-stack inventory management system to provide a single source of truth for stock. The application, built with Next.js, offers a clean interface for performing complete CRUD operations on inventory items. It tracks essential business data points like item location, brand, and cost. By leveraging a real-time Firebase database, the system ensures that stock levels are always accurate and instantly updated, helping to streamline operations and reduce costly errors.

The Problem

Standard note-taking apps aren't optimized for the unique needs of language learners, who often need to organize vocabulary, grammar rules, and practice sentences in a structured, easily searchable way.

My Solution

I designed and built LearnNote, a specialized note-taking platform for students of new languages. The application is built on a modern Next.js and TypeScript stack for type safety and scalability. To ensure user data is secure and private, I implemented robust user authentication using Clerk. All notes are stored and managed in a Firebase database, which allows for powerful and instantaneous search functionality, enabling users to find the exact note or lesson they need in seconds.
